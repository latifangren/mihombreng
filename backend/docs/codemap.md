# backend/docs/

## Responsibility
Auto-generated Swagger 2.0 API documentation produced by `swaggo/swag` from Go source annotations. Provides machine-readable (JSON + YAML) and machine-embeddable (Go source) representations of the Mihombreng REST API schema for use by Swagger UI, code generators, and API testing tools.

## Design
Three output files generated by `swag init -g cmd/server/main.go -o docs`:

| File | Format | Purpose |
|------|--------|---------|
| `docs.go` | Go source | Embeds the full Swagger JSON spec as a Go string constant (`docTemplate`); registers it with `swag.Register()` for runtime Swagger UI serving |
| `swagger.json` | JSON | Machine-readable OpenAPI 2.0 spec for import into Postman, Insomnia, Swagger UI, etc. |
| `swagger.yaml` | YAML | Human-readable equivalent of swagger.json |

All three files describe the same API surface. The Go file uses `swag.Spec` struct with `SwaggerTemplate` containing `{{marshal}}` and `{{escape}}` template functions. API metadata: title "Mihombreng API", version "1.0", basePath "/api/v1", MIT license.

## Flow
```
Go source annotations (@Summary, @Description, @Tags, @Param, @Response)
  └── swag init -g cmd/server/main.go -o docs
        ├── docs.go       (Go embed template + SwaggerInfo registration)
        ├── swagger.json  (OpenAPI 2.0 JSON)
        └── swagger.yaml  (OpenAPI 2.0 YAML)
```

## Integration
- **Source**: Generated from `//swag` annotations in `cmd/server/main.go` and handler files under `internal/http/handler/`
- **Tool**: `swaggo/swag` CLI (`swag init`)
- **Build**: `scripts/build.sh` stage [3/5] regenerates these files before each build
- **Runtime**: `docs.go` registers with `swag.Register()` for optional Swagger UI serving (`api.enable_swagger: false` in `mihombreng.yaml`)
- **Consumer**: `docs/02-api-reference.md` is the hand-written companion with richer examples; these files are the machine-readable canonical source
